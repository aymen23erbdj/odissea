무들 1.5.3 판 (2005년 11월 11일)

(본 배포판은 중요한 보안오류를 교정한 것으로, 무들의 이전판을 이용하시어 사이트를 운영하시는 분들께서는 빠른 시일안에 본 1.5.3판으로 업그레이드하시기를 강력히 권하는 바입니다.)

새로 첨가된 것들

  • SCORM 1.3 (SCORM 2004) 지원!
  • 새로운 Single-Sign-On API 기능 가능

오류 수정

일반

  • 이미 알려진 PHP 설정의 보안문제가 무들 설정에 포함되어 있을 때 경고를 함
  • 최근 보고된 보안 문제를 바르게 고침(security.moodle.org) 참조)
  • 중급/대형 사이트에서의 좀 더 빨라진 백업. 백업 부분에서의 많은 문제점들을 바르게 교정함
  • 리눅스/유닉스 시스템에서 디렉토리 크기의 출력이 상당히 빨라졌고, Admin->Variables 로부터 "path_to_du" 로 설정할 수 있게 됨으로서 성능을 제대로 발휘할 수 있게 됨
  • 프록시 서버 뒤에서 서버나 클라이언트가 작동하고 있을 때에도 ip 추적을 할 수 있게 되어 의미있는 기록을 남김
  • 약간의 PHP 가속기의 문제를 교정함
  • Upgrade scripts have been refined and work much better for PostgreSQL installations
  • General PostgreSQL port code cleanup
  • Upgrading to 1.5.3 will fix any legacy Journal to Online Assignment upgrade issues
  • RSS libraries now support RSS 2.0 categories, see bug 3654
  • Better accesibilty in file uploads, see bug 3662
  • Better support for site-wide HTTPS, see bug 3848
  • We now send away bots (like Googlebot) from that try mangled URLs, see bug 3958
  • Better DST support for non-logged-in users
  • Better DST support for repeat events in Calendar
  • Small fixes to the lesson, choice and grade modules
  • Minor CSS improvements to formal_white and orangewhite themes
  • Better pagination in course listing and loglive pages
  • Fixed an IE-only bug related to HTMLArea content where user-edited tables could float and hide part of the UI.

인증

  • Fixes to change password/forgotten password with external auth
  • Data fetched from external DB or LDAP is now truncated correctly
  • Fixed bug 4305 -- better login block behaviour when using secureforms
  • Better support for utf-8 user data from external auth
  • LDAP: General fixes covering sync_users script and logging of errors
  • LDAP: Fixed bug 3141 - Can't update external data with LDAP authentication
  • LDAP: Fixed bug 3992 - LDAP password including a quote does not work - credits go to Kita
  • LDAP: Better support for ActiveDirectory.
  • LDAP: Fixed bug #3594
  • LDAP: Better handling of multi-source field mapping
  • LDAP: Fix for using DN as idnumber - thanks fo Jeff Graham - http://moodle.org/mod/forum/discuss.php?d=28840

등록 및 메타코스

  • Metacourses: better support for MySQL v3.23.x

  • LDAP: Course auto-creation works again
  • LDAP: Fixed several bugs reported by Jeff Graham and Barron Koralesky -- including bugs 3912 and 3974
  • LDAP: We can now bind non-anonymously to LDAP
  • Authorize.net: several fixes

자원

포럼

  • Big and small optimizations to tracking of read/unread messages, specially for PostgreSQL

  • Some more actions are now supported by backup/restore when processing log records, see bug 3582
  • Now message forum RSS feeds are including category data -- useful under some RSS agregators able to group messages, see bug 3654
  • Fixed paging on forum search

블록

퀴즈

  • Fixed potential data data corruption bug 3915

  • Fixed bug 3807: Question numbering issue
  • Fixes for the item analysis plug-in, assembled by Jean-Michel
  • Now we reset the timeout counter to avoid timeouts
  • Several improvements contributed by Jean-Michel Vedrine. You can now change the number of attempts per page displayed on screen, and download results to Excel/text file, including detailed grades
  • Fixed export bug with non-English languages

SCORM

  • Fixed a bug with next and prev SCO search

  • Censor filter: Added original word as title to the blacked-out version. The filter can obscure actual important words by mistake (eg Dickens) and there needs to be some way to recover the meaning

무들 1.5 판 (2005년 6월 5일)

주요 기능

  • 웹페이지가 XHTML Transitional 1.0에 호환됨
  • WAI (W3C), SENDA (영국) 및 Section 508 (미국) 기준의 호환을 목표로 한 접속가능성의 향상
  • CSS를 통하여 계단형 테마, 사용자 테마, 배움터 테마 등을 매 페이지마다 세밀하게 조정 가능한 매우 강력한 새 테마 시스템
  • 사이트내 모든 사용자간 직접 연락이 가능한 실시간 팝업창, 안내, 에메일 전송, 블록, 활동기록, 위즈윅 에디터 등이 통합된 메시지 기능
  • 한 눈에 새 게시물을 알아볼 수 있고 화면 설정을 다양하게 조절할 수 있게 한 강조 가능한 포럼 추적기능(포럼, 개인, 사이트 별 읽음/안읽음 표시)
  • 다양한 블록 및 좀더 향상된 블록 설정을 가능하게 한 새로운 블록 시스템
  • 전 세계 어디에서나 모든 일정에 대해 개인의 시간대에 맞게 표시되게 하는 일광 절약 시간제 완벽 지원
  • 각기 다른 활동에 대해 가중치를 달리 부여하거나 (개별적으로 적용할 수 있는) 특별 부가점 설정 및 향상된 화면 표시, 정열 기능을 갖춘 새롭고 확장된 성적표
  • 사이트 정책을 수용하도록 사용자에게 요구가능

기타 전반적인 향상 기능

관리

  • 새 "관리 모드"를 통한 관리자가 사이트의 작동을 불가능하게 할 수 있음(예를 들어 업그레이드 시)
  • moodle.org 나 기타 사이트로 부터 새로운 시간대를 자동으로 전송하여 데이터베이스에 저장 가능(Olson 파일 지원)
  • (개별적으로 사용가능한) 새 판의 무들 MySQL관리 모듈
  • 새로운 달력 옵션페이지를 통한 개별적인 달력 인터페이스 설정 제공

인증

  • 새 Shibboleth 지원
  • 새 CAS 지원
  • 새 PAM 지원
  • 향상된 LDAP 통합 기능을 통하여, 배움터 생성 및 모둠 과제를 조정 가능. 사용자 동기화 및 작동 성능 향상
  • 외부 인증시스템을 사용할 때 관리자가 사용자의 특정 항목을 잠글 수 있음
  • 향상된 세션 핸들링을 통하여 PHP 설정과정에서 오류원인이 될 수 있는 "중첩" 세션을 검출 및 처리 가능
  • 세션이 데이터베이스에 저장 될 수 있음(한꺼번에 설치하는데 좋음!)
  • 사용자들에게 비밀번호를 강제로 바꾸도록 요청 가능

파일

  • 모든 영역에서 파일 처리 과정을 표준화
  • 향상된 업로드 기능을 통하여 새 문서에 대해 ClamAV를 사용한 자동 바이러스 점검 기능 지원
  • Slasharguments가 IIS(ISAPI에 대해서는 PHP 4.3.11로 업그레이드 요망)상에서도 작동

필터

  • 문서의 검색 및 강조에 필요한 새로운 필터의 일상적인 작성을 지원하는 필터 라이브러리의 표준화
  • Glossary필터와 같은 복잡한 필터에 대한 대폭적인 효율성 향상
  • (선택적으로) 전 사이트의 모든 사용자가 입력한 문서를 삭제하거나 이들을 유효한 XTML 코드로 전환할 수 있는 새로운 Tidy 필터로 W3C Tidy 프로그램 사용
  • 개선된 censorship 필터로 스타일을 사용하여 빈칸을 제거하고 언어팩의 단어 목록을 사용
  • 향상된 플래쉬 MP3 플래이어로 다운로드 및 작동시 진행과정을 살필 수 있음
  • 빠르고 구문오류에 좀 더 융통성이 있으며 새로운 편집 구문<span lang="en"></span>을 이용할 수 있는 향상된 다중 언어 필터 채용
  • 인터넷 사전, 활동 등에서 사용되는 자동 연결 필터는 (예전에 쓰던) 모든 경우의 연결에 작동할 뿐만 아니라, 각 문서 블록 혹은 전체 페이지에 대해서도 작동($CFG->filtermatchonepertext, $CFG->filtermatchoneperpage).
  • 무들내에 기재된 활동명, 표제 및 기타 작은 문장 등 좀더 많은 문서를 필터링 할 수 있음. 이를 통해 사용자 선택 언어 상황하에서도 무들의 완벽한 다중언어 사이트 구현 가능

테마

  • 모듈을 통한 표준 형식 제공
  • 필요한 다른 테마의 형식을 채용 가능
  • 관리자가 허용할 시, 사용자 및 배움터의 개별적인 테마 채택 가능
  • 다수의 페이지에 대해 탭인터페이스의 적합화(Implementation)
  • 모듈, 블록, 언어에 대해 표준 스타일 정의 가능

HTML 편집기

  • 편집 도구창의 도구버튼들은 관리자에 의해 설정될 수 있음
  • 편집 영뎍내의 문서 찾아 고치기 가능(정규 표현의 선택적 사용을 활용)

  • 새로운 메타=코스를 통해 일단 메타코스와 연결된 배움터에 등록을 한 사람은 일반 배움터(메타코스)에 자동으로 등록될 수 있도록 허용함
  • 배움터간 내용을 복사할 수 있는 새 도구 생김

새 블록

  • 원격 RSS 전송: 외부 뉴스 채널을 무들 배움터 내부로 가져와 표시 가능
  • 인터넷 사전 선택 : 배움터 주화면의 무들 인터넷 사전으로부터 내용을 선별하여 표시할 수 있음

달력

  • 모듬을 사용하는 다인수 배움터가 있는 사이트에서의 속도 향상
  • 반복되는 일정을 개별적 혹은 동시에 삭제하거나 고칠 수있음
  • 새로운 "기억 필터 설정"을 통해 달력 필터들이 로그인시 상태를 기억할 수 있게 함

활동 모듈 향상

과제

  • 새롭게 학급을 토대로한 설계로 완벽하게 재구성함으로서 새 플러그인 과제 유형을 가능하게 함
  • 과제를 파일로 받지 않는 새로운 온라인 문서 과제는 등급을 매길 때 직접 평을 써넣을 수 있으며 이는 예전의 저널 모듈을 효과적으로 대치할 수 있음
  • 다인수 학습을 다루기 위한 대폭 향상된 성적 인터페이스 제공
  • 학생이 늦게 제출하는 것을 막을 뿐만 아니라 새 과제를 제출했을 때 교사에게 이메일로 공지하도록 새로운 설정 옵션 제공

대화

  • 대화창이 부드럽게 보이며 심지어 선택적인 작동 서버 데몬 없이도 빠르게 작동함
  • 대화창에 블럭(예: 부가적 정보 등)이 첨부될 수 있음

간이 설문(Choice)

  • 선택지의 수에 제한이 없도록 재 작성됨
  • 일련의 옵션에 "기표"할 수 있게 하는 도구로 사용할 수 있도록 선택지당 사용자의 수를 제한할 수 있음
  • 설문 결과를 XLS 나 TXT 파일 형태중 선택하여 내려받을 수 있게 하였음

포럼

  • 강력하고 새로운 구글 형태의 포럼 탐색 도구
  • 포럼 기사의 읽음/읽지않음 추적 - 읽지 않은 메시지는 배움터 화면, 포럼 페이지, 토론 목록 및 토론 보기에서 강조되어 표시됨
  • 교사 혹은 사용자가 추적 기능을 켜거나 끌 수 있음
  • 사용자 신상명세 화면에서는 그 개인의 토론은 물론 모든 게시물을 볼 수 있음
  • 관리자가 사용자의 메시지를 수정할 때는 안내글이 첨부됨
  • 토론 목록에는 각 토론에 마지막으로 게시한 사용자를 보여주며 그 글로 직접 연결해 줌
  • 모둠 허용 포럼에서는 토론 목록은 모둠 요약에 링크됨과 동시에 모둠의 글타래를 보여줌

인터넷 사전

  • 새로운 설정 기능을 통하여 각 인터넷 사전에 대한 인쇄미리보기 기능을 가능/불가능 하게 설정할 수 있음
  • 새로운 검색 시스템을 통해 어느 곳에 있는 단어라도 검색할 수 있음(정확한 구문 일치 대용)
  • 활동 추적을 할 수 있게 좀 더 자세한 정보가 로그에 기록됨

저널(Journal)

  • 저널 모듈은 1.4파 이후로 변하지 않았으며 이제 새로운 기능(Online Assignment)으로 대체될 것입니다. 업그레이드를 통해 여러분의 모든 저널활동을 온라인 과제로 변환시킬 것이며, 모든 기존의 저널활동을 감출 것입니다. 만일 이를 원하지 않는다면 config.php 파일 내에 다음과 같이 선언해 주기 바랍니다 : $CFG->noconvertjournals = true;

학습

  • 시간 제한형 학습 제공
  • (성적이 저장되지 않는) 예습 과정 생성 가능
  • 학습도중 학생들이 획득한 점수를 확인할 수 있는 옵션 제공
  • 학생들이 학습을 완료하기 전, 자신들의 답안을 검토할 수 있도록 허용
  • 새로운 슬라이드 쇼 모드(단 슬라이드로서 부속된 표만 보여짐)
  • 새로운 왼쪽 차림표 (가능하게 한 부속 테이블에 대해서만)
  • 학습이 암호로 보호될 수 있음
  • 수업 만들기 화면을 가지 형태로 볼 수 있음
  • 학생들이 자신의 최고점을 게시할 수 있음
  • 동일 배움터 내에서 기존의 학습 설정을 기본설정으로 저장할 수 있는 새로운 옵션 생김
  • 학생의 시도들을 삭제할 수 있음
  • 새로운 페이지 이동 :
    • 앞 페이지로 이동
    • 가지 내에서 풀지 않은 문제로 이동
    • 가지 내에서 무작위 질문 문항으로 이동
    • 클러스터 내에서 풀지 않은 문제로 이동
  • 학습 통계 보기 기능 첨가
  • 새로운 문제 유형 : 에세이. 교사들은 각 에세이를 검토하고 평을 쓸 수 있으며, 등급을 매기고 학생들에게 통지할 수 있음
  • 새로운 문형 제작 인터페이스

퀴즈

  • 동일한 퀴즈내에서 문제를 반복적으로 시도할 수 있게 하거나 학생들 답에 대한 반응을 변경함으로서 시의 적절한 문형을 다룰 수 있음
  • 학생들이 정답을 찾을 때까지 동일한 퀴즈 내에서 한 문제를 반복해서 즉각적으로 풀어볼 수 있도록 허용 가능
  • 어떤 문제에 대하여 틀렸을 경우 일정한 비율로 감점을 하는 벌점 기능 도입
  • RQP 웹 서비스 프로토콜을 지속적으로 무들 퀴즈 내부로 통합하기 위해 문항의 작성과 채점을 외부적으로 처리할 수 있게 함(예: 대수 평가 엔진)
  • IMS QTI 문항들을 처리할 수 있게 준비함으로써 이러한 문제들을 웹에서 활용할 수 있게 함
  • 퀴즈 미리보기, 편집, 다시 보기 기능을 위한 새로운 교사용 책갈피형 인터페이스
  • 새 "향상된 보안" 모드를 이용하여 브라우저의 기능을 통제 함으로써 구분된 전화면으로 퀴즈를 볼 수 있음
  • 여러 페이지에 걸쳐 학생들에게 퀴즈가 제시될 수 있음. 교사에 의해 한 페이지에 제시되는 문항수가 조절됨
  • 퀴즈 화면에 (결과표 혹은 부가적 참고 자료 등을 보여주는) 블록이 첨가될 수 있음
  • 데이터베이스내의 문제들을 좀더 세련되고 깔끔하게 선택 및 관리할 수 있는 인터페이스 제공. 선택된 문항들을 한꺼번에 퀴즈에 첨가, 다른 항목으로 이동, 삭제가 가능
  • 무작위 문항들은 주어진 버튼을 통해 퀴즈 문제 목록에서 첨가할 수 있음 Random questions are now added to the quiz question list with a dedicated button, leaving a cleaner interface at questions database without phantom placeholders
  • 최종 성적 결과는 미리 교사가 정한 유효숫자범위 내에서 실수로 표시될 수 있음
  • 사용자 표시 선택 기능을 가진 향상된 결과 페이지를 통해 깔끔하게 정렬된 원하는 항목만을 볼 수 있음
  • 문항들은 향상된 퀴즈미리보기 기능 및 문항미리보기 화면을 통해 클릭 한 번으로 편집 가능
  • 한번 클릭으로 문항을 복사할 수 있게하여 손쉽게 문항을 수정할 수 있게 만듬
  • 문항 항목 구조 및 편집 인터페이스를 고침
  • 항목이 또 다른 하위 중첩 항목을 포함할 수 있으며 재 정열 가능
  • 문항 편집 페이지에서 항목에 속한 문제냐 혹은 그 하위 항목을 보여줄 것이냐를 선택하는 기능이 가능해짐
  • 단지 상위 항목의 무작위 문제를 첨가할 것이냐 혹은 상위항목에서 하위 항목을 선택할 것이냐를 결정하는 것이 가능해짐
  • 학생들이 답을 작성한 이후에는 퀴즈의 문항 묶음의 변경을 못하도록 문항 생성 기능 조정
  • 중복/중첩된 문항의 승인 및 거절 기능 향상
  • 퀴즈 편집 화면에서 긴 퀴즈 항목의 문항 표시를 페이지 별로 표시되도록 함
  • 새로운 IMS QTI (2.0) 및 xhtml 출력 방식 채용. 여타 방식의 버그 수정 및 개선
  • 새로운 도입 유형인 쟁점 퀴즈 도입. 다른 문제 유형의 버그 수정 및 개선

Scorm

  • 사용자 결과 자료의 저장 및 보고 기능이 향상됨
  • AICC 패키지를 불러들일 수 있음

  • 백업/복구 및 기타 자질구레한 기능 향상을 통한 위키의 신뢰성 제고
